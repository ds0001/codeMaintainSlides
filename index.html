Maintainable Javascript: Nicholas Zakas

Based on the presentation given at Fluent 2012 and his book, Maintainable Javascript.

Why should we care?
  • we spend most of our time maintaining code
  • we code in a team environment
  • most likely you won't be maintaining your code
maintainable code is...
  • adhering to a consistent style
  • human readable
  • using well known programming conventions
  • clear & consistent in its naming conventions
  • providing clear & consistent comments
  • commented where hacks or lesser known techniques are used

event handling guidelines
  • event handlers should only handle events
  • don't pass the event object around
  • properly separated event handling
event handling guidelines 
      //the wrong way!!!
      function handleClick(event)   {
          var popup = document.getElementById("popup");

          popup.style.left = event.clientX + "px";
          popup.style.top = event.clientY + "px";
          popup.className = "reveal";
      }
event handling guidelines 
     //a better way
     function handleClick(event){
        showPopup(event.clientX, event.clientY);
     }

     function showPopup(x,y) {
        var popup = document.getElementById("popup");

        popup.style.left = x + "px";
        popup.style.top = y + "px";
        popup.className = "reveal";
    }
general guidelines
  • dont modify objects you don't own
  • avoid global functions and variables
  • throw your own errors
  • avoid null comparisons
  • separate config data
dont modify objects you don't own 
        //don't add new methods
        Array.prototype.awYeah = function () {
            alert("Awyeah!");
        };
        
        //don't override methods
        YUI.use = function () {
            alert("Awyeah!");
        };
throw your own errors 
          var Controller = {
              addClass : function (element, className) {
                  if (!element) {
                      throw new Error("addClass: 1st arg missing.");
                  }
                  element.className += "" + className;
              }
          };
avoid null comparisons
  • A problematic pattern in JavaScript is testing a variable against null, presumably to determine whether the variable has been filled in with an appropriate value.
  • use instanceof to test for specific object types
  • use typeof to test for primitive type (string, object, etc.)
separate config data
  • All URLs neeed by the app
  • strings displayed to users
  • html that needs to be create by app
  • settings
  • repeated unique values
  • any value that may change in the future
